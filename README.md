# Мерч - Система учёта товаров

Веб-приложение для управления товарами, поступлениями и продажами магазина одежды с многопользовательским доступом.

## Возможности

### Основные функции
- **Дашборд** с графиками продаж, статистикой и аналитикой
- **Учёт товаров** с категориями, брендами, размерами и автоматическим расчётом остатков
- **Журнал поступлений** с автоматическим пересчётом остатков при добавлении/удалении
- **Журнал продаж** с корзиной товаров, разными типами цен и отслеживанием продавцов
- **Планы продаж** с визуальным контролем выполнения
- **Фильтры и поиск** по всем таблицам с возможностью сброса

### Дополнительные возможности
- **Многопользовательский режим** с ролями (Владелец / Продавец)
- **Редактирование** всех записей (товары, поступления, продажи, планы)
- **Адаптивный дизайн** для работы с телефона, планшета и компьютера
- **Мобильное меню** с быстрым доступом к разделам
- **Переключатель тем** (тёмная / светлая) с сохранением выбора
- **Welcome-экран** для удобной навигации после входа
- **Корпоративный стиль** с логотипом и фирменными цветами

## Быстрый старт

### Шаг 1: Создание Firebase проекта

1. Перейдите на [Firebase Console](https://console.firebase.google.com/)
2. Нажмите "Добавить проект"
3. Назовите проект (например, `merch-store`)
4. Отключите Google Analytics (не требуется для данного приложения)
5. Нажмите "Создать проект"

### Шаг 2: Настройка Firestore Database

1. В левом меню выберите **Firestore Database**
2. Нажмите "Создать базу данных"
3. Выберите **"Запустить в тестовом режиме"** (для начала работы)
4. Выберите локацию (например, `eur3 (europe-west)`)
5. Нажмите "Включить"

### Шаг 3: Настройка Authentication

1. В левом меню выберите **Authentication**
2. Нажмите "Начать"
3. Перейдите на вкладку **Sign-in method**
4. Включите метод **Электронная почта/пароль**
5. Нажмите "Сохранить"

### Шаг 4: Получение конфигурации Firebase

1. В настройках проекта (⚙️) выберите **"Настройки проекта"**
2. Прокрутите вниз до раздела **"Ваши приложения"**
3. Нажмите на иконку **Web** (</>)
4. Зарегистрируйте приложение (название любое, например `merch-web`)
5. Скопируйте объект `firebaseConfig`

### Шаг 5: Настройка проекта

1. Клонируйте репозиторий или скачайте файлы
2. Откройте файл `index.html`
3. Найдите блок с `firebaseConfig` (примерно строки 180-190)
4. Замените значения на свои из скопированной конфигурации:

```javascript
const firebaseConfig = {
    apiKey: "ваш-api-key",
    authDomain: "ваш-project.firebaseapp.com",
    projectId: "ваш-project-id",
    storageBucket: "ваш-project.appspot.com",
    messagingSenderId: "ваш-sender-id",
    appId: "ваш-app-id",
    measurementId: "ваш-measurement-id"
};
```

### Шаг 6: Создание первого пользователя (Владельца)

Поскольку публичная регистрация отключена, первого пользователя нужно создать вручную:

**Вариант A: Через Firebase Console**

1. В Firebase Console перейдите в **Authentication → Users**
2. Нажмите "Добавить пользователя"
3. Введите email и пароль для будущего владельца
4. Скопируйте **UID** созданного пользователя

5. Перейдите в **Firestore Database**
6. Создайте коллекцию `users` (если её нет)
7. Добавьте документ с ID равным скопированному UID
8. Заполните поля:
   ```javascript
   {
     uid: "скоpированный-uid",
     name: "Имя Владельца",
     email: "email@example.com",
     role: "owner",
     createdAt: "2026-07-05T00:00:00.000Z"
   }
   ```

**Вариант B: Временно включить регистрацию**

Если нужно быстро создать первого пользователя:
1. Временно добавьте форму регистрации обратно в `index.html`
2. Зарегистрируйтесь как Владелец
3. Удалите форму регистрации из кода

### Шаг 7: Деплой на GitHub Pages

1. Создайте публичный репозиторий на GitHub (например, `merch-store`)
2. Загрузите все файлы в репозиторий:

```bash
git init
git add .
git commit -m "Initial commit"
git branch -M main
git remote add origin https://github.com/ваш-username/merch-store.git
git push -u origin main
```

3. В настройках репозитория перейдите в **Settings → Pages**
4. В разделе **Source** выберите:
   - **Deploy from a branch**
   - Branch: `main`
   - Folder: `/ (root)`
5. Нажмите **Save**
6. Через 1-2 минуты сайт будет доступен по адресу: `https://ваш-username.github.io/merch-store/`

### Шаг 8: Настройка безопасности (Важно!)

1. В Firebase Console перейдите в **Authentication → Settings → Authorized domains**
2. Добавьте домен вашего GitHub Pages (например, `ваш-username.github.io`)

3. Перейдите в **Firestore Database → Rules** и настройте правила безопасности:

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    // Только авторизованные пользователи могут читать/писать
    match /{document=**} {
      allow read, write: if request.auth != null;
    }
    
    // Дополнительные ограничения можно добавить по необходимости
    match /users/{userId} {
      allow read: if request.auth != null;
      allow write: if request.auth != null && 
        get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'owner';
    }
  }
}
```

## Использование системы

### Вход в систему

1. Откройте сайт по ссылке GitHub Pages
2. Введите email и пароль
3. После входа появится Welcome-экран с приветствием
4. Выберите нужный раздел из карточек быстрого доступа или используйте меню сверху

### Добавление сотрудников (только для Владельца)

1. Перейдите в раздел **Пользователи**
2. Нажмите **+ Добавить сотрудника**
3. Заполните форму:
   - Имя сотрудника
   - Email (будет использоваться для входа)
   - Пароль (минимум 6 символов)
   - Роль (Продавец или Владелец)
4. Нажмите **Создать сотрудника**
5. Передайте сотруднику его email и пароль

### Разграничение доступа

**Владелец** имеет полный доступ ко всем разделам:
- Дашборд (полная аналитика)
- Товары (добавление, редактирование, удаление)
- Поступления (добавление, редактирование, удаление)
- Продажи (добавление, редактирование, удаление)
- Планы продаж (создание, редактирование, удаление)
- Пользователи (создание и удаление сотрудников)

**Продавец** имеет ограниченный доступ:
- Дашборд (базовая аналитика)
- Товары (просмотр, редактирование, удаление)
- Поступления (добавление, редактирование, удаление)
- Продажи (добавление, редактирование, удаление)
- Планы продаж (просмотр)

## Основные операции

### Работа с товарами

**Добавление товара:**
- Товары → + Добавить товар
- Заполните форму (обязательно только название)
- Артикул генерируется автоматически
- Сохраните товар

**Редактирование товара:**
- Найдите нужный товар в таблице
- Нажмите кнопку **Изменить**
- Внесите изменения и сохраните

**Фильтрация товаров:**
- Используйте фильтры сверху таблицы (поиск, категория, пол, бренд, размер, цена)
- Нажмите **Сбросить** для очистки всех фильтров

### Приёмка товара (Поступления)

1. Перейдите в раздел **Поступления**
2. Нажмите **+ Добавить поступление**
3. Выберите существующий товар или создайте новый (кнопка ➕ Добавить новый товар)
4. Укажите количество
5. Нажмите **Сохранить поступление**
6. Остатки автоматически пересчитаются

### Регистрация продажи

1. Перейдите в раздел **Продажи**
2. Нажмите **+ Добавить продажу**
3. Добавляйте товары в корзину:
   - Выберите товар из списка
   - Нажмите **+ Добавить в корзину**
   - Можно добавить несколько разных товаров
4. Для каждого товара в корзине настройте:
   - Количество
   - Тип цены (Оригинал / По скидке / Другая цена)
5. Проверьте итоговую сумму
6. Нажмите **Оформить продажу**
7. Остатки автоматически спишутся

### Работа с планами продаж

1. Перейдите в раздел **Планы** (только для Владельца)
2. Нажмите **+ Установить план**
3. Заполните форму:
   - Название периода (например, "Июль 2026")
   - Дата начала
   - Дата окончания
   - План продаж в рублях
4. Нажмите **Сохранить план**
5. Система автоматически будет отслеживать выполнение плана

## Дашборд и аналитика

### Доступные метрики

**Продажи:**
- Продажи за период (настраиваемый: 7/30/90 дней или свой период)
- Количество продаж
- Максимальная продажа

**Средний чек:**
- Средний чек за период
- Медианный чек
- Минимальный чек

**Склад:**
- Всего товаров на складе
- Последнее поступление
- Всего позиций в каталоге

**Планы продаж:**
- Текущий план с прогресс-баром выполнения
- Последний завершённый план (если есть)

**Графики:**
- Топ товаров (5 или 10 позиций)
- Динамика продаж (с выбором периода)
- Распределение товаров (по категориям, цене или полу)

### Настройка периодов

Для большинства метрик можно выбрать:
- **7 дней** - последняя неделя
- **30 дней** - последний месяц
- **3 месяца** - последний квартал
- **Свой период** - указать произвольные даты начала и окончания

## Решение проблем

### Не удаётся войти в систему

**Возможные причины:**
- Неверный email или пароль
- Пользователь не существует в Firebase Authentication
- Отсутствует запись в коллекции `users` в Firestore
- Домен не добавлен в Authorized domains

**Решение:**
1. Проверьте консоль браузера (F12) на наличие ошибок
2. Убедитесь, что пользователь создан в Firebase Console → Authentication
3. Проверьте, что в Firestore есть документ в коллекции `users` с правильным UID
4. Добавьте домен сайта в Authorized domains

### Не отображаются данные на дашборде

**Возможные причины:**
- Данные ещё не загружены из Firestore
- Ошибки в правилах безопасности Firestore

**Решение:**
1. Обновите страницу
2. Проверьте, что в правилах Firestore разрешено чтение для авторизованных пользователей
3. Откройте консоль браузера и проверьте наличие ошибок

### Ошибки при сохранении данных

**Возможные причины:**
- Недостаточно прав (попытки удаления для Продавца)
- Проблемы с подключением к Firebase
- Ошибки в правилах безопасности

**Решение:**
1. Проверьте роль текущего пользователя
2. Убедитесь, что есть подключение к интернету
3. Проверьте консоль браузера на наличие ошибок
4. Пересмотрите правила безопасности в Firestore

## Рекомендации по безопасности

1. **Не храните конфиденциальные данные** в клиентском коде
2. **Используйте правила безопасности** Firestore для ограничения доступа
3. **Регулярно проверяйте** список пользователей в Firebase Console
4. **Устанавливайте сложные пароли** для всех пользователей
5. **Не публикуйте** API ключи в публичных репозиториях (хотя Firebase API ключи безопасны для клиента)
